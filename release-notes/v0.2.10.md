## Phosphene Electron v0.2.10

Web publish working-directory fix. This patch prevents Wrangler from trying to write deploy cache files at the filesystem root when Phosphene is launched from Finder, Dock, or Spotlight.

### Changes
- **Web publishing** - runs Cloudflare Pages deploys from the generated publish parent directory while deploying the `site` directory, so Wrangler uses a writable `.wrangler/cache` location outside the uploaded site assets.

### Validation
- `npm run test:node -- electron/web-publish/deployer.test.ts` - targeted deployer regression passing
- Manual Cloudflare Pages deploy from the generated publish bundle - deployment completed successfully
- `npm test` - all tests passing
- `npm run lint -- --max-warnings=0` - zero warnings
- `npm run build` + `npm run build:main` + `npm run build:electron` - all succeed
- `npm run test:e2e` - smoke tests passing
