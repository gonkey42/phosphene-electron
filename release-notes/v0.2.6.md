## Phosphene Electron v0.2.6

Board-pack automation release. This version adds the generic import substrate that lets tools and agents generate Phosphene workspaces and boards as files, then import them through the CLI or the app.

### Changes
- **Board pack format** — adds a versioned Board Pack v1 folder contract with manifest and board-file validation.
- **Importer and CLI** — imports generic board-pack folders into SQLite-backed workspaces and boards, copies referenced pack assets into Phosphene's image store, and exposes `npm run board-pack:import` for headless automation.
- **App integration** — adds board-pack IPC, preload/platform bridge support, and a File menu import path for running app sessions.
- **Automation testing** — adds a tiny generic board-pack fixture and e2e coverage that verifies imported workspaces and boards appear after relaunch.
- **Dev inspection** — adds development-only localhost remote debugging through `PHOSPHENE_DEBUG_PORT`.
- **Test stability** — pins the backup test clock and preserves remote image response bytes when constructing downloaded image files.

### Validation

- `npm run lint -- --max-warnings=0` — zero warnings
- `npm run build` + `npm run build:main` + `npm run build:electron` — all succeed
